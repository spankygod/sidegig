# Raket Mobile

Expo app for Google sign-in and poster workflows against the Fastify API in `../backend`.

## Environment

Create `mobile/.env.local` from `mobile/.env.local.example`.

Required variables:

```bash
EXPO_PUBLIC_API_URL=http://10.0.2.2:3000
EXPO_PUBLIC_SUPABASE_URL=https://YOUR_PROJECT.supabase.co
EXPO_PUBLIC_SUPABASE_PUBLISHABLE_KEY=YOUR_SUPABASE_ANON_KEY
```

Use the API URL that matches your runtime:

- Android emulator: `http://10.0.2.2:3000`
- iOS simulator: `http://127.0.0.1:3000`
- Physical device: `http://<your-lan-ip>:3000`

## Supabase auth setup

1. Enable Google in Supabase Auth.
2. In Supabase redirect URLs, allow `mobile://auth/callback` or `mobile://**`.
3. If you test through Expo Go, also allow the callback URL generated by Expo for your current session.

The app exchanges the Supabase session for backend access by sending the Supabase access token as `Authorization: Bearer <token>` to the Fastify API.

## Run

```bash
cd mobile
npm install
npx expo start
```

Run the backend separately from `../backend`.

## What is implemented

- Google sign-in through Supabase
- Secure persisted session in Expo
- Authenticated calls to `/v1/auth/me`, `/v1/users/me`, and `/v1/gigs/mine`
- Poster flow for creating gigs with draft or published state
- Basic profile screen and sign-out
